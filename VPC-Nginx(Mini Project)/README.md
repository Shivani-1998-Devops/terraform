# VPC Nginx Mini Project

## Project Overview

This Terraform project creates a simple AWS infrastructure for an NGINX web server inside a private VPC setup. It includes:
- A VPC (`aws_vpc`) with CIDR `10.0.0.0/16`
- A public subnet (`aws_subnet`) with CIDR `10.0.1.0/24`
- An Internet Gateway (`aws_internet_gateway`)
- A public route table and subnet association (`aws_route_table`, `aws_route_table_association`)
- A security group allowing SSH (22), HTTP (80), and HTTPS (443)
- An EC2 instance in the public subnet with a public IP

> Note: The EC2 instance AMI ID is currently hard-coded as `ami-0880bf9508ddc7bab` for the AWS region `ap-south-1`. Replace this AMI with a current, supported one if needed.

## Files

- `provider.tf` - Terraform configuration for the AWS provider and required versions.
- `vpc.tf` - VPC networking resources, subnet, IGW, route table, and association.
- `ec2.tf` - EC2 instance and security group resources.
- `terraform.tfstate` / `terraform.tfstate.backup` - Terraform state files generated by previous runs.

## Prerequisites

1. Install Terraform `>= 1.7.0` and `< 2.0.0`.
2. Configure AWS CLI credentials or use environment variables.
3. Ensure you have access to the AWS account and region `ap-south-1`.

### AWS credentials setup

```bash
export AWS_ACCESS_KEY_ID="your_access_key_id"
export AWS_SECRET_ACCESS_KEY="your_secret_access_key"
export AWS_DEFAULT_REGION="ap-south-1"
```

Or on Windows PowerShell:

```powershell
$env:AWS_ACCESS_KEY_ID = "your_access_key_id"
$env:AWS_SECRET_ACCESS_KEY = "your_secret_access_key"
$env:AWS_DEFAULT_REGION = "ap-south-1"
```

## Step-by-step Usage

### 1. Initialize Terraform

Initialize the Terraform working directory and download the AWS provider.

```bash
cd "d:/Devops/Teraform Project/git-push-repo/VPC-Nginx(Mini Project)"
terraform init
```

### 2. Format Terraform files (optional but recommended)

```bash
terraform fmt
```

### 3. Validate the configuration

```bash
terraform validate
```

### 4. Review the execution plan

Generate and inspect the plan before applying.

```bash
terraform plan -out=tfplan
```

### 5. Apply the Terraform plan

Create the AWS resources.

```bash
terraform apply "tfplan"
```

If you prefer a one-step command:

```bash
terraform apply
```

### 6. Verify the deployment

After `terraform apply`, Terraform will output the created resources. To verify manually:

- Open the AWS Console.
- Confirm the VPC, subnet, internet gateway, route table, security group, and EC2 instance exist.
- Check the EC2 instance public IP and test HTTP access.

## Cleanup

Destroy all resources created by this project when no longer needed.

```bash
terraform destroy
```

## Important Notes

- The security group allows inbound SSH and HTTP traffic from anywhere (`0.0.0.0/0`). Restrict this for production use.
- The EC2 instance uses a public IP address and is placed in a public subnet.
- Confirm the AMI ID is valid for the chosen AWS region. If the AMI is outdated or unavailable, update the `ami` value in `ec2.tf`.
